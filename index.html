Chunk-First API Surface Recovery from JavaScript Bundles

Jia Shuo Yang · Independent Researcher · Published July 1, 2026 · DOI: 10.5281/zenodo.21106612

Download PDF Zenodo record GitHub repository ORCID

Abstract

Modern single-page applications often place API client code outside the JavaScript files directly visible from the initial HTML response. Lazy route chunks, bundler runtime tables, dynamic imports, and framework manifests can hide endpoint evidence from URL extractors that only scan the entry corpus. This paper presents Chunk-First API Surface Recovery (CF-ASR), a reproducible static-analysis workflow that expands the production JavaScript corpus before endpoint extraction and emits endpoint-oriented records with source provenance, method hints, dynamic path skeletons, and conservative endpoint-only filtering.

Across 15 reviewed production-mode application benchmarks, deterministic CF-ASR recovers strict endpoint paths with precision 0.94 and recall 0.92. A separately scored bounded model-assisted review for composed endpoints raises recall to 0.95. A request-shape study finds visible path, query, body, or header evidence for 251 of 650 review-confirmed recovered endpoint paths, showing that parameter evidence is useful but less complete than endpoint recovery. Recovered records are intended for authorized triage and dynamic testing, not as proof that endpoints are live, reachable, authorized, or vulnerable.
