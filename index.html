Research overview

Chunk-First API Surface Recovery from JavaScript Bundles

CF-ASR studies how backend API evidence can be recovered from modern production JavaScript bundles. The research examines why code splitting causes entry-point analysis to miss client-side endpoint evidence and explores a chunk-first approach that gathers observable application code before analyzing its API surface.

J. S. Yang · Independent Researcher

GitHub repository ORCID

Problem

Modern web applications can place API-related code in lazy or deferred JavaScript that is absent from the initial page load.

Research idea

Recover the observable production JavaScript corpus first, then examine that expanded corpus for backend endpoint evidence.

Purpose

Provide a more complete and reviewable starting point for authorized API-surface analysis and follow-on testing.

About the research

Production frontend applications are commonly divided into an initial bundle and additional code loaded only when users visit particular routes or features. As a result, an analysis limited to the scripts visible at entry time may observe only part of the client-side API surface.

CF-ASR investigates a different ordering: first recover the observable JavaScript belonging to the deployed application, including additional chunks, and then analyze the resulting corpus for backend-facing endpoint evidence. The research evaluates whether this ordering provides a stronger basis for understanding the API surface represented by the production frontend.

Research boundary

The work focuses on static, frontend-visible endpoint evidence for authorized security research. It does not determine whether an endpoint is live, accessible, authorized, or vulnerable.
