CF-ASR: Chunk-First API Surface Recovery from JavaScript Bundles

CF-ASR studies a simple problem with practical impact: modern frontend applications often hide backend API evidence in lazy-loaded JavaScript chunks that are invisible from the initial HTML response. The paper presents a chunk-first static-analysis workflow that expands the production JavaScript corpus before endpoint extraction and then reports conservative, auditable endpoint findings with source provenance.

Jia Shuo Yang · Independent Researcher · Published July 1, 2026 · DOI: 10.5281/zenodo.21106612

Problem

URL-entry scans undercount API evidence when endpoint-bearing code lives in hidden route chunks and deferred bundle fragments.

Method

Recover initial scripts, discover additional chunks, beautify the expanded corpus, extract backend evidence, and apply strict endpoint filtering.

Output

Endpoint-oriented records with source provenance, method hints, dynamic path skeletons, and conservative filtering boundaries.

Scope

This page publishes the paper only. Code and the reproducibility environment are not included in this GitHub Pages release.

Abstract

Modern single-page applications often place API client code outside the JavaScript files directly visible from the initial HTML response. Lazy route chunks, bundler runtime tables, dynamic imports, and framework manifests can hide endpoint evidence from URL extractors that only scan the entry corpus. CF-ASR expands the production JavaScript corpus before endpoint extraction and emits endpoint-oriented records with source provenance, method hints, dynamic path skeletons, and conservative endpoint-only filtering.

The paper argues that chunk-first acquisition materially improves endpoint recovery over URL-entry baselines because those baselines frequently miss dormant JavaScript that is never requested during an entry crawl. It also studies a bounded model-assisted review stage for composed endpoints and discusses the limits of request-shape recovery relative to endpoint-path recovery.

Research scope

  • CF-ASR focuses on recovering frontend-visible backend API surface evidence from production JavaScript bundles.
  • The method expands hidden and lazy-loaded chunks before endpoint extraction, rather than treating the entry response as the full client-side corpus.
  • The paper separates deterministic endpoint recovery from bounded model-assisted composed-endpoint review.
  • Recovered findings are intended for authorized analysis and testing workflows, not as proof that an endpoint is live, reachable, authorized, or vulnerable.
